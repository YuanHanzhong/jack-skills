# ADS/DWS 双轨文档统一模板

> 📌 本文件是所有ADS和DWS文档的标准模板·Claude创建新文档时必须读取本文件
> 📌 两种文档类型用同一个模板，区别只是写入目标和顶部类型标记
> 📌 [占位符] = Claude根据对话内容填入·框架结构不可更改
> 📌 每个区块注释：谁维护、格式要求、与其他区块的一致性关系

---

## 双轨分类规则（CRITICAL·每次创建文档前必看）

```
📚 学习内化类 → 默认写入ADS看板
  特征：理解/认知/思维模式/价值观/反思/成长/为什么/底层逻辑
  目的：焊忠长期看，直到学会。配合闪卡测试/内化。
  生命周期：长期存在 → 学会后归档

🔧 工程实践类 → 默认写入DWS
  特征：优化技能/修改代码/重构/修复/实现/部署/自动化/配置
  目的：实施阶段焊忠监控进度。断点续传+进度跟踪。
  生命周期：实施中 → 完成后归档

⚠️ 焊忠说了算：
  - Claude判断类型后推荐，焊忠可以指定放哪里
  - 焊忠说放ADS就放ADS，说放DWS就放DWS
  - Claude只善意提醒一次，不阻拦不纠缠
  - 两个区域都能写，不硬性限制

⚠️ 模板完全一样：
  - 区块结构、格式规范、一致性检查全部相同
  - 唯一区别：文档顶部类型标记行不同
```

## 区块关系图（一致性检查依据·每次写入前必看）

```
结论 ←→ 行动（结论变→行动必须同步变）
结论 ←→ 断点续传·关键上下文（结论变→上下文描述必须一致）
洞察 → 闪卡知识点（新洞察→评估是否新增L1知识点）
思维链·💡新发现 → 闪卡知识点（新发现→必须评估是否值得记为知识点）
思维链·💡新发现 → 进展记录（新发现→追加一行进展）
进展记录 ←→ 断点续传（进展变→断点必须同步更新）
📊进度 ←→ 断点续传（进度变→断点必须同步更新）
指挥台To-Do ←→ 📊进度（To-Do变→进度必须同步映射）
指挥台 ←→ 所有区块（指挥台=最高标准，所有内容朝此目标对齐）
ADS状态字段 ←→ 📊进度（状态变→进度反映最新状态）
备注字段 ←→ 文档核心信息（备注是扫描时唯一可见摘要，必须精准）
```

## To-Do树形映射规则（CRITICAL）

```
焊忠指挥台 To-Do = 树的根节点 + 树枝（中间节点）
Claude 只做叶子节点 = 最细粒度的执行步骤

示例：
指挥台（焊忠写）：
  - [ ] 完成技能优化        ← 根节点（焊忠写）
    - [ ] 改模板             ← 树枝（焊忠写）
    - [ ] 改代码             ← 树枝（焊忠写）

📊 进度中（Claude映射）：
  ⏳ 完成技能优化                     ← 原样复制·和指挥台完全一致
    ⏳ 改模板                          ← 原样复制·和指挥台完全一致
      ⏳ 更新ads-template.md [auto]    ← 叶子（Claude拆解）
      ⏳ 同步区块关系图 [auto]          ← 叶子（Claude拆解）
    ⏳ 改代码                          ← 原样复制·和指挥台完全一致
      ⏳ 重写build_ads_document [auto]  ← 叶子（Claude拆解）

规则：
  ① 焊忠写的节点（根+枝）→ Claude原样复制到📊进度，文字完全一致
  ② Claude只在叶子位置追加[auto]标记的子步骤
  ③ 焊忠打勾 → Claude检测到后：父级+所有子叶子全标✅
  ④ 没有焊忠To-Do时 → Claude按自己判断生成默认进度项
```

## 扫描铁律（数据库级扫描 vs 页面级读取·CRITICAL）

```
⛔ 扫描数据库（ADS看板）时：只读属性字段（标题/状态/优先级/备注/日期），不读页面内容
⛔ ADS看板会非常大 → 扫描必须走Notion筛选（状态/最近编辑时间），禁止全量fetch内容
⛔ 归档 = ADS看板内改状态为「📦 归档」+ 用Notion筛选视图，不是搬家
✅ 只有焊忠明确说「读这个文档」「看一下XXX」「打开XXX」时，才fetch页面内容
✅ 备注字段 = 扫描时唯一可见的文档摘要，所以每次写入/更新必须同步更新备注
✅ 扫描根目录时：只读页面标题和子页面列表，不读页面内容（除非有子页面需要列出）
```

## 备注字段写法规范（CRITICAL·扫描时唯一可见摘要）

```
格式：[当前阶段] 核心结论/状态 | 最近进展 | 下一步
示例：[探索中] 优化本能双刃剑·自驱vs焦虑 | 已确立3条铁律 | 下一步测试L2
长度：50-100字，必须一眼看懂文档当前状态
更新时机：每次结论/阶段/状态变化时同步更新备注字段
```

## 触发闪卡知识点新增的信号

```
信号1：焊忠说出"哦原来是这样"/"对！"/"明白了"等顿悟瞬间
信号2：焊忠的认知发生变化（之前认为A→现在认为B）
信号3：对话推导出新的行为准则/铁律
信号4：发现底层机制（如"掌控感是最底层需求"）
信号5：两个已知概念产生新连接（如"矛盾#4和对话偏好同源"）
→ 标记 💡 **【关键词】** + 评估是否新增闪卡 + 追加进展记录
```

## 闪卡知识点格式规范（必须严格遵守·learning-engine按此格式读取）

```
格式：
📌 **【知识点名称】** | `掌握度状态` | 上次测试：[日期或-] | 连续答对：[N] | [L级别]
> > 💡 挖词依据：[为什么是知识点·关键词·关联·挖空方向]

挖词依据写法规范：
  1. 说明核心考点在哪里
  2. 标注关键词（L1出挖空题时挖的词）
  3. 写明挖空方向：[上下文]→____（答：[关键词]）
  4. 标注关联（与哪个矛盾/铁律/洞察相关）

示例：
📌 **【借来的信条=排异反应】** | `未测试` | - | 0 | L1
> > 💡 挖词依据：焊忠对"直接指出问题"本能抵触，跟矛盾#4同源。关键词=排异反应。挖空：别人给的结论不管多对，只要不是自己走到的→____（答：执行不了/排异反应）

📌 **【优化本能双刃剑】** | `未测试` | - | 0 | L2
> > 💡 挖词依据：同一个能力两种输出。关键词=自驱vs焦虑驱动。反向推导：焊忠废寝忘食磨工具时，判断是____驱动还是____驱动（答：自驱/心流 vs 焦虑/逃避）

L级别规则：
  L1·概念理解 = 考「是什么」→ 题型=挖空填充（挖关键词让用户填）
  L2·多角度应用 = 考「为什么」→ 题型=反向推导/场景变体/对比辨析
  L3·生活场景活用 = 考「遇到了能不能做出来」→ 题型=真实情境题

本技能只维护：新增知识点 + 挖词依据 + L级别 + 挖空方向
learning-engine维护：掌握度/分数/测试时间/连续答对（本技能不改这4字段）
```
