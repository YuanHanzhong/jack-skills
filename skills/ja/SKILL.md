## Context

- Current working directory: !`pwd`
- STARTT template path: `.claude/skills/notion-writer/templates/startt_template.md`
- Sessions directory: `03_DWS/sessions/`

## Your task

**使用 AI 大模型（你自己）**智能分析当前会话，提取关键信息并填充到 STARTT 模板，生成高质量的会话记录。

⚠️ **关键原则**：不使用代码/正则表达式提取，而是用 AI 理解整个对话上下文后智能总结。

### 执行步骤

#### Step 1: 获取 JSONL 会话数据（增强性，失败不阻断）

运行会话数据提取工具，获取被上下文压缩丢失的早期消息：

```bash
bun run archive-conversation --json 2>/dev/null
```

> **注意**：不使用 `--full` 参数。`--full` 会输出完整消息内容（40KB+），超过 Bash 工具的 30000 字符截断限制，导致 JSON 不完整无法解析。默认模式每条消息截断到 2000 字符，对 AI 沉淀已足够。

- ✅ 成功：将 JSON 数据作为**补充数据源**，用于后续融合分析
- ⚠️ 失败：跳过此步骤，仅依赖对话上下文（仍可生成文档）

#### Step 2: 获取 Git 提交历史（增强性，失败不阻断）

获取当天的 Git 活动，补充具体的代码变更信息：

```bash
git log --oneline --since="today" 2>/dev/null
git diff HEAD --stat 2>/dev/null
```

- ✅ 成功：将提交记录和文件变更作为**实施细节数据源**
- ⚠️ 失败：跳过此步骤

#### Step 3: 读取 STARTT 模板

使用 Read 工具读取模板文件：`.claude/skills/notion-writer/templates/startt_template.md`

#### Step 4: 三源融合智能分析

> **AI 决策**: 基于三个数据源综合分析会话内容，提取结构化知识
>
> **输入**:
> 1. 当前对话上下文（最高优先级）— Claude 自身记忆，最新最准
> 2. JSONL 数据（补充）— 被压缩的早期消息，补充完整时间线
> 3. Git 提交历史（补充）— 具体的代码变更和提交信息
>
> **处理（融合规则）**:
> - 主题/结论 ← 对话上下文优先
> - 思维链 ← 对话上下文为主线，JSONL 补充早期探索过程
> - 实施细节 ← git log 提供文件变更列表和提交消息
> - 冲突时以对话上下文为准
>
> **输出（供 Step 6 使用）**:
> 1. **会话主题** - 用简洁描述性的标题概括（非"未命名会话"）
> 2. **结论** - 一句话总结 + 3-5 个核心要点
> 3. **问题描述** - 背景、具体问题、期望效果
> 4. **思维链** - 意图确认 → 分析探索 → 实施验证（融合 JSONL 中的早期探索）
> 5. **闪卡** - 可复习的关键知识点（表格形式）
> 6. **待落实** - 可复用的行动模板（表格形式）
> 7. **核心洞察** - 3-5 条关键认知
> 8. **关联资源** - 相关文档和链接
> 9. **实施记录** - 从 git log 提取的文件变更和提交消息（如有）

#### Step 5: 确定分类和元数据

> **AI 决策**: 基于会话内容确定文档的分类归属和元数据
>
> **输入**: Step 4 的分析结果（会话主题、核心内容）
> **处理**: 匹配知识分类体系，提取关键词
> **输出（供 Step 6 使用）**:
> - **分类**：项目开发 / 知识学习 / 系统配置 / 问题调试 / 其他
> - **标签**：5-8 个关键词
> - **Front Matter**：session 对象（id, date, datetime, category, tags, status）

#### Step 6: 生成并保存文档

> **AI 决策**: 基于 Step 4 和 Step 5 的分析结果，填充 STARTT 模板
>
> **输入**: Step 4 的提取内容 + Step 5 的分类元数据 + Step 3 的模板结构
> **处理**: 将分析结果填充到模板各章节，确保：
> - Front Matter 格式正确（YAML）
> - 所有章节都有实质内容（不留"待补充"）
> - 闪卡和待落实使用表格格式
> - 思维链完整体现推导过程（含早期探索）
> - 标题有意义（非"未命名会话"）
> - 如有 git 变更，在文档中体现实施记录
>
> **输出**: 完整的 STARTT 格式文档内容

**执行**: 使用 Write 工具保存文档

- 文件名格式：`YY-MMDD-HH-主题名称.md`
- 保存路径：`03_DWS/sessions/<分类>/`
- 例如：`03_DWS/sessions/项目开发/26-0215-10-修复skill逻辑漏洞.md`

#### Step 7: 验证文档质量

使用 Read 工具读取 Step 6 保存的文档，逐项检查：

- [ ] 标题是否有意义（非"未命名会话"）
- [ ] 所有章节是否填充完整
- [ ] Front Matter 格式是否正确
- [ ] 文件命名是否合规（`YY-MMDD-HH-主题名称.md`）

**STARTT 完整格式检查**（>10 轮对话时必须通过）：
- [ ] `## 🔍 对话轮次` 存在，且有至少一个 `### Round-N`
- [ ] `## 🗓️ 进展跟踪` 存在，且有至少一条日期记录
- [ ] `## 🔁 下次启动点` 存在，且包含"当前进度"和"下次第一步"
- [ ] 如缺失上述任一章节 → **补充完整后再保存，不允许跳过**

如有不合格项，修正后重新保存。
